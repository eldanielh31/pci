Simics Python Release Notes

1 Introduction

This document identifies important information for the Simics Python package for Simics 6. All users of this specific package should review this document carefully.

2 Changes Since Version 6.0.0

The following is a list of changes since the initial 6.0.0 release.

  • 6.0.2 (build 6311)

    • Common
      • The signing certificate on Simics packages has been updated.

  • 6.0.1 (build 6306)

    • Python
      • The Simics embedded Python interpreter has been upgraded to version 3.10.14.
    • mini-python
      • Fixed a problem where mini-python failed to run on Windows machines with restrictive execution policies.

  • 6.0.0 (build 6291)

    • Common
      • Simics no longer comes with documentation in PDF format.
      • Documentation in HTML format has been added.
      • Fixed a problem on Windows where the installer still asked for the Model Builder key (bug #SIMINT-1236).
      • Support for Python 2.7 has been removed. Simics now only embeds a Python 3 interpreter.
      • The separation of source code into separate packages has been removed. Source code is now included directly into each package.
      • The installers for Linux and macOS have been updated. Support for fallback mode of decryption keys has been removed.
    • Python
      • The Simics embedded Python interpreter has been upgraded to version 3.10.13.
      • The Simics embedded Python interpreter has been upgraded to version 3.10.12.
      • The Simics embedded Python interpreter has been upgraded to version 3.9.18.
      • The Python library lxml has been removed from the Simics Base package, for security reasons (bug #SIMICS-20978).
      • Simics now distributes pyi files of the core Python modules.
      • Simics no longer distributes pypredef files of the core Python modules.
      • The Simics embedded Python interpreter has been upgraded to version 3.9.16.
      • The Simics embedded Python interpreter has been upgraded to version 3.9.15.
      • The Python library now includes pip and setuptools, so that pip can be used with mini-python (bug #HSD-14018026828).
      • Python deprecation warnings are now turned on, unless Simics is started with -wno-deprecated. If Simics is started with -werror then the deprecation warnings result in Python exceptions (bug #SIMICS-20274).
      • Python warning filters can now be controlled using the SIMICS_PYTHONWARNINGS environment variable. The supported usage is the simple all warning unconditional override described at https://docs.python.org/3/using/cmdline.html#envvar-PYTHONWARNINGS.
      • Fixed a problem in the Tcl/Tk library included with Python on Windows.
      • Python upgraded to Python 3.9 due to Python 3.7 is going EOL mid-2023. Required changes to user Python code may include not distributing .pyc files and updates to valid Python 3.9 code.
      • On Windows, the Simics embedded Python interpreter has been upgraded to version 3.7.13. The linked zlib library has been upgraded to 1.2.13.
      • The Python library now includes the psutil module.
      • The Python library now includes the ipyxact module.
      • The Python library now includes the sqlite3 module (bug #HSD-14017070185).
      • The Python library on Windows now includes the Tcl/Tk library, so that the tkinter module can be used.
      • The Simics embedded Python interpreter has been upgraded to version 3.7.12.
      • The Simics embedded Python interpreter has been upgraded to version 3.7.11.
      • The Simics embedded Python interpreter has been upgraded to version 3.7.9.
      • Removed numpy from the Simics Python distribution.
      • The Simics embedded Python 2 interpreter has been upgraded to version 2.7.16, on Windows.
      • The Simics embedded Python 3 interpreter has been upgraded to version 3.7.4, on Windows.
      • The numpy package that is distributed with the Simics Python 3 interpreter has been upgraded to version 1.17, on Windows (bug #HSD-2208178272).
      • Fixed a problem where the wrong OpenSSL was used by the Simics Python 2 interpreter on Linux (bug #HSD-1507350606).
      • The Simics embedded Python 2 interpreter has been upgraded to version 2.7.16, on Linux and macOS.
      • The Simics embedded Python 3 interpreter has been upgraded to version 3.7.4, on Linux and macOS.
      • Simics now supports Python 3.7.2 as its builtin Python interpreter. The default is still Python 2.7 and this will be supported until its EOL, which is 1st of January 2020.
    • mini-python
      • On Windows, the mini-python binary now respects long paths if enabled on the system; in particular, if the registry key HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\FileSystem\LongPathsEnabled is set to 1, then the binary can run a Python file whose path exceeds 260 characters (bug #HSD-22018805048).
      • On Linux, the mini-python binary can now be used with the venv and pip Python modules to install extra libraries (bug #HSD-18023192821).

5 Detailed List of Limitations

This section describes in detail the known limitations of the Simics Python package. Please refer to section 4 for a more general description.

A register or field marked as Not implemented is present with read-write semantics but has no side effects on simulation. A register marked as Not implemented (design limitation) has the same semantics as a "Not implemented" register and there is no plan to extend the model with this functionality. A register with Lack of documentation has not been implemented because there is no available documentation describing its semantics.
